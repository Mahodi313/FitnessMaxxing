## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase REST/PostgREST `.rpc()` | Untrusted RPC payload + UUID + cursor + metric + since parameters cross this boundary; PostgREST routes to the SQL function which executes with caller's auth context (security invoker) |
| anon-jwt user-session → workout_sessions + exercise_sets + workout_plans tables | RLS policies on these tables scope rows to the calling user_id; new RPC functions inherit |

|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information disclosure | get_session_summaries RPC | mitigate | `security invoker` + RLS on workout_sessions scopes to caller's user_id; `test-rls.ts` Phase 6 extension asserts User A's call returns no rows containing User B's sessB.id (Task 3) |

    ```sql
    create or replace function public.get_exercise_top_sets(
      p_exercise_id uuid,
      p_since timestamptz,
      p_limit int default 10
    )
    returns table (
      session_id uuid,
      completed_at timestamptz,
      weight_kg numeric,
      reps int
    )
    language sql
    security invoker
    stable
    set search_path = ''
    as $$
      select session_id, completed_at, weight_kg, reps
      from (
        select distinct on (es.session_id)
          es.session_id,
          es.completed_at,
          es.weight_kg,
          es.reps
        from public.exercise_sets es
        inner join public.workout_sessions s
          on s.id = es.session_id
         and s.finished_at is not null
        where es.exercise_id = p_exercise_id
          and es.set_type = 'working'
          and (p_since is null or es.completed_at >= p_since)
        order by es.session_id, es.weight_kg desc, es.completed_at desc
      ) tops
      order by completed_at desc
      limit p_limit;
    $$;

    revoke all on function public.get_exercise_top_sets(uuid, timestamptz, int) from public;
    grant execute on function public.get_exercise_top_sets(uuid, timestamptz, int) to authenticated;
    ```
