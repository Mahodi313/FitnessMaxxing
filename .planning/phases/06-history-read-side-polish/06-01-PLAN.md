<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Supabase REST/PostgREST `.rpc()` | Untrusted RPC payload + cursor parameter cross this boundary; PostgREST routes to the SQL function which executes with caller's auth context (security invoker) |
| anon-jwt user-session → workout_sessions + exercise_sets + workout_plans tables | RLS policies on these tables scope rows to the calling user_id; new RPC functions inherit |
| AsyncStorage (offline-cache) → Reactive UI | TanStack persister hydrates cached infinite-query data on cold-start; UUIDs in cache are non-PII per T-06-07 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information disclosure | get_session_summaries RPC | mitigate | `security invoker` + RLS on workout_sessions scopes to caller's user_id; `test-rls.ts` Phase 6 extension asserts User A's call returns no rows containing User B's sessB.id (Task 3) |
| T-06-02 | Information disclosure | get_exercise_chart RPC | mitigate | `security invoker` + RLS on exercise_sets + workout_sessions parent-FK; `test-rls.ts` Phase 6 extension asserts User A's call with User B's exercise_id returns empty (Task 3) |
| T-06-03 | Tampering | workout_sessions DELETE cross-user | mitigate | RLS `using ... user_id = (select auth.uid())` on workout_sessions blocks server-side; `test-rls.ts` Phase 6 extension asserts assertWriteBlocked when User A attempts to DELETE User B's session (Task 3) |
| T-06-04 | Spoofing | exercise_id URL tampering in future chart route | mitigate | Same RLS chain on exercise_sets returns empty rows for spoofed exercise_id; Plan 03 surfaces empty-state. Test in Task 3 RLS extension. |
| T-06-05 | Tampering | p_cursor parameter | mitigate | RPC signature types p_cursor as timestamptz so Postgres rejects non-parseable strings with 4xx; future cursor → empty result via WHERE clause |
