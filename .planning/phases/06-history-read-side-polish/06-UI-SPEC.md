Generated by gsd-ui-researcher; to be verified by gsd-ui-checker.

> **shadcn note:** shadcn/ui is a React-DOM component library and is incompatible with React Native runtimes. This project uses NativeWind 4 (Tailwind 3 class names compiled to RN styles) per the locked stack in CLAUDE.md and per the Phase 3/4/5 UI-SPECs. **Chart canvas height:** `style={{ height: 240 }}` (explicit pixel height ‚Äî Victory Native XL `<CartesianChart>` requires a measured height; 240px = 60 √ó 4 so it stays on the 4-pt grid). Falls inside the standard chart-block (between toggle-row and "Senaste 10 passen"-section). Card-container around the chart uses default `md` horizontal padding so the visual canvas width = device-width ‚àí 32.

**Touch target minimum:** All interactive elements ‚â• 44√ó44pt per Apple HIG (matches Phase 3/4/5 contract).
- History-list row: `py-4` (16px) wrapping a multi-line content view ‚Üí ~64pt total. Same convention as Phase 4 plan-list row.
- Chart-ikon on plan_exercise row (D-24, Pitfall 6.1): Pressable wrapping a 22pt `<Ionicons name="stats-chart">` with `p-3` (12px each side) = 46pt total. Lives in `flex-row justify-between` row-end; tap-bubble does not propagate to outer row (`Pressable` siblings are independent ‚Äî explicit `onPress` swallow).
- Segmented-control segment: `flex-1 py-2 px-3` minimum per segment ‚Üí height ~36pt visually, but combined with the parent control's `py-1` (4px) and the segment's content `text-base` (16pt) the total tappable region is ‚â•44pt vertical because the parent's `py-1` is inside the Pressable's slop. To guarantee on-grid, every segment Pressable wraps `<Text>` with `py-2` (8px) inside a parent `p-1` (4px) container ‚Üí 8+16+8 = 32pt content + 4pt outer = 36pt visible, with `hitSlop={{ top: 4, bottom: 4 }}` ‚Üí 44pt effective.
- Delete-pass action in session-detail header-right: header-right `<Pressable>` with `p-2` (8px) wrapping a 22pt `<Ionicons name="ellipsis-horizontal">` = 38pt + `hitSlop={6}` ‚Üí 50pt. Tap opens the inline-overlay-confirm (NOT a modal-portal per Phase 4 commit `e07029a`).
- Confirmation-overlay buttons ("Avbripa" / "Ta bort"): `py-4` (16px) + `text-base` (16pt) ‚Üí ~48pt. Same primary-CTA dimensions as Phase 3/4/5.
- Chart-press tap-state (Victory Native `useChartPressState`): no hit-target threshold ‚Äî the entire chart canvas surface (240pt √ó full-width) is the tap surface; the press-state finds the nearest data point automatically.
- Empty-state CTA on chart route (when 0 points): no CTA exists in V1 (text-only empty state per D-17); user discovers chart by logging more sets via the active-workout flow.

**`gap-2` vs `mb-2` for list-row separation:** Phase 6 follows Phase 4 convention ‚Äî history-list and session-detail card-lists use `mb-2` per item (not `gap-2` on parent `<View>`) because `FlatList` does not support `gap` directly. Lists use `ItemSeparatorComponent={() => <View className="h-2" />}` where ergonomic.

Exceptions: none ‚Äî all values are 4-multiples.

The callout itself (Skia `<RoundedRect>` + `<Text>` group) is positioned via `chartPressState.x.position` / `chartPressState.y.y.position` (Reanimated shared values).

**Date-formatting decision (Claude's Discretion ‚Üí resolved here):** Use date-fns `format(date, 'd MMM yyyy', { locale: sv })`. Swedish locale lowercase month abbreviations: `14 maj 2026`, `3 dec 2026`. The full-year is included to disambiguate when soak-test reaches year boundary (Jan 2027). Time-of-day is **omitted** from the row to reduce visual noise; users who want session start time tap into `/history/[sessionId]` (where the header title shows the date and the summary-header shows `duration`).

**Row click target:** Entire `Pressable` (whole row) routes to `/history/[sessionId]` per D-05. The Pressable boundary is the rounded card; no chevron on the right because right-column already shows metadata. (Differs from Phase 4 plan-list-row which has a chevron; rationale: plan-list-row's right-edge is empty space so chevron adds an affordance hint; history-row's right-edge is metadata so a chevron would crowd. The whole-row-pressable pattern is well-established in iOS.)

**Plan-name fallback rendering (D-08):** If `plan_id IS NULL` (cascade from deleted plan, rare), render the secondary text as `‚Äî ingen plan` (em-dash + space + "ingen plan"). ### Loading / cold-start

- **History-list cold cache:** `usePersistedQueryClient` from Phase 4 hydrates `sessionsKeys.listInfinite()` from AsyncStorage synchronously. The user sees the last-cached page within ~100ms. If cache is empty (first launch post-clear), `useInfiniteQuery` enters `isPending` and the list area shows a centered `<ActivityIndicator size="large" color={accent} />` (matches Phase 4 cold-load).
- **Session-detail cold cache:** Tap from history-list ‚Üí `useSessionQuery(id)` is seeded by `initialData: () => queryClient.getQueryData(sessionsKeys.listInfinite())?.pages.flat().find(s => s.id === id)` (Phase 4 commit `eca0540` pattern). No loading-flash. If user deep-links directly to `/history/<id>` (route-restoration on cold start), `usePersistedQueryClient` first hydrates the list cache, then `useSessionQuery` finds the seed. Worst-case fallback: centered `Laddar‚Ä¶` Body Muted text.
- **Chart route cold cache:** `useExerciseChartQuery(exerciseId, metric, window)` has no list-cache seed (chart query key is unique per metric+window combo). First load shows `Laddar‚Ä¶` in the chart-area; toggle switches reuse cached data via TanStack's `keepPreviousData: true` so the UI never flickers to empty during a window-swap. After the first successful fetch, subsequent toggles to previously-fetched (metric, window) combos are instant (cache-hit) and re-fetches happen in background.
- **No skeleton-loaders in V1** ‚Äî `Laddar‚Ä¶` text is the convention (D-12). Skeleton loaders are V1.1 polish if text-only loading reads as broken.

### Offline state surfaces

- **Global OfflineBanner** ‚Äî visible on (tabs)/history when offline; inherited from Phase 4 D-05. Phase 6 does not touch.
- **History-list works offline (ROADMAP success #4):** `usePersistedQueryClient` hydrates the infinite-query cache from AsyncStorage at app start. User sees the last-fetched pages without n√§t. Pull-to-refresh fires a refetch which fails silently offline (`networkMode: 'offlineFirst'`); user remains on the cached data. The OfflineBanner communicates the state.
- **Session-detail works offline** if the session is in cache (which it is when navigating from list ‚Äî `initialData` pattern). Direct deep-link to a never-cached session-id while offline: shows `N√•got gick fel. F√∂rs√∂k igen.` because `useSessionQuery` cannot fetch and has no seed. Acceptable ‚Äî this is a rare path.
- **Chart works offline** if the (exerciseId, metric, window) combo is in cache from a prior online session. If never cached, the empty-state renders (`Inga pass √§n f√∂r den h√§r √∂vningen`); user toggles to another window or comes back online. Acceptable ‚Äî chart is read-side polish, not a must-have for offline UX.
- **Delete-pass mutation when offline:** `useDeleteSession.mutate` queues per `networkMode: 'offlineFirst'`. Optimistic onMutate removes the session from `sessionsKeys.listInfinite()` cache + `sessionsKeys.detail(id)` cache. UI feedback: user is routed back to history-list (`router.replace`) + sees `Passet borttaget` toast; the session is gone visually. When connectivity returns, `resumePausedMutations()` flushes the delete to Supabase. RLS + FK on delete cascade handle the server-side cleanup. No "queued for sync" badge per row (F24 V2-deferred).
